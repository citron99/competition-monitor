# AI-мониторинг конкурентов (мультимодальное приложение)

Персональный AI-анализатор рынка конкурентов, который умеет работать с текстом, изображениями и веб‑страницами.  
Проект реализован на FastAPI, ProxyAPI (OpenAI-совместимый API), Selenium и PyQt6.

## Цель кейса

Автоматизировать первичный маркетинговый анализ конкурентов в выбранной нише.  
В моём случае ниша — **продажа аккумуляторов для автопогрузчиков в России**: локально проработаны реальные сайты российских поставщиков и сервисных компаний, торгующих аккумуляторами и комплектующими для погрузочной техники.

## Локально проработанные конкуренты

Для тестирования и настройки ассистента были использованы реальные конкуренты на рынке РФ (тематика: «продажа аккумуляторов для автопогрузчиков»):

- сайты поставщиков тяговых аккумуляторов для вилочных погрузчиков;
- сервисные компании по обслуживанию и замене аккумуляторов;
- производители и дилеры аккумуляторных батарей для складской техники.

Точные URL и названия компаний не выкладываются в README по этическим причинам, но добавлены в `.env`/`config.py` в виде списка `COMPETITOR_URLS` и использовались для парсинга и анализа.

## Основные возможности

- Анализ текстов конкурентов (лендинги, описания, PDF‑отчёты в текстовом виде):
  - выделение сильных и слабых сторон;
  - поиск уникальных предложений (УТП);
  - рекомендации по улучшению;
  - дополнительные метрики: `design_score` и `animation_potential` (0–10).

- Анализ изображений (скриншоты лендингов, баннеры, упаковка):
  - детальное описание изображения;
  - маркетинговые инсайты;
  - оценка визуального стиля (`visual_style_score`);
  - дополнительные метрики по дизайну и анимации.

- Парсинг сайтов конкурентов через Selenium:
  - автоматическая загрузка страницы;
  - извлечение `title`, первого `h1` и первого осмысленного абзаца;
  - создание скриншота страницы;
  - анализ текста + анализ скриншота;
  - сохранение результатов в историю.

- История запросов:
  - хранение последних запросов (текст, изображение, парсинг);
  - краткое резюме ответа (по тексту и картинке).

- Desktop‑обёртка (PyQt6 + PyInstaller):
  - отдельный `.exe`‑файл `competitionmonitor.exe` для Windows (надстройка над backend‑API).

## Технологический стек

- Backend:
  - FastAPI — REST API для анализа конкурентов;
  - ProxyAPI / OpenAI‑совместимый API — текстовые и визуальные модели;
  - Pydantic — схемы запросов/ответов;
  - Selenium + ChromeDriver — парсинг сайтов и скриншоты.

- Desktop:
  - PyQt6 — GUI‑клиент для работы с ассистентом;
  - PyInstaller — сборка `competitionmonitor.exe`.

- Прочее:
  - `.env` для конфиденциальных настроек;
  - логирование через стандартный `logging`.

## Структура проекта (кратко)

- `backend/`
  - `main.py` — FastAPI‑приложение и маршруты;
  - `config.py` — настройки (ProxyAPI, парсер, список URL конкурентов и т.д.);
  - `models/schemas.py` — Pydantic‑схемы (включая `design_score`, `animation_potential`);
  - `services/openaiservice.py` — работа с ProxyAPI (анализ текста, изображений, скриншотов сайтов);
  - `services/parsingservice.py` — Selenium‑парсер + скриншоты.

- `desktop/`
  - `main.py` — PyQt6‑клиент (GUI) для работы с backend‑ом.

- Корень:
  - `run.py` — запуск FastAPI сервера (мониторинг конкурентов);
  - `build.py` — сборка desktop‑приложения через PyInstaller;
  - `data/` — тестовые материалы конкурентов (PDF/изображения/скриншоты);
  - `history.json` — сохранённая история (опционально).

## Настройка окружения

1. Клонировать репозиторий:

   ```bash
   git clone https://github.com/<your-nick>/pem08-competition-monitor.git
   cd pem08-competition-monitor
   ```

2. Создать и активировать виртуальное окружение.

3. Установить зависимости:

   ```bash
   pip install -r requirements.txt
   ```

4. Создать файл `.env` в корне (по примеру):

   ```env
   PROXY_API_KEY=YOUR_PROXY_API_KEY
   OPENAI_MODEL=gpt-4o-mini
   OPENAI_VISION_MODEL=gpt-4o-mini

   # список конкурентов (реальные сайты в РФ по продаже аккумуляторов для автопогрузчиков)
   COMPETITOR_URLS=https://primer1.ru,https://primer2.ru,https://primer3.ru
   ```

   `.env` добавлен в `.gitignore` и не выкладывается в репозиторий.

## Запуск backend (API)

1. Запуск FastAPI сервера:

   ```bash
   python run.py
   ```

2. После старта доступны:
   - веб‑интерфейс: `http://localhost:8000`
   - Swagger UI: `http://localhost:8000/docs`
   - ReDoc: `http://localhost:8000/redoc`

3. Основные эндпоинты:
   - `POST /analyzetext` — анализ текста конкурента;
   - `POST /analyzeimage` — анализ изображения (base64);
   - `POST /parsedemo` — демонстрация парсинга сайтов конкурентов + анализ;
   - `GET /history` — история запросов.

## Сборка desktop‑приложения

1. Убедиться, что установлены PyQt6 и PyInstaller:

   ```bash
   python -m pip install PyQt6 PyQt6-Qt6 PyQt6-sip pyinstaller
   ```

2. Запустить скрипт сборки:

   ```bash
   python build.py
   ```

3. После успешной сборки:
   - exe‑файл появится в папке `dist/` под именем `competitionmonitor.exe`.

4. Запуск desktop‑клиента:

   ```bash
   cd dist
   .\competitionmonitor.exe
   ```

   Backend (`run.py`) должен быть заранее запущен.

## Типовой сценарий использования

1. Собрать данные конкурентов (для ниши «аккумуляторы для автопогрузчиков в РФ»):
   - сохранить PDF‑каталоги, коммерческие предложения, скриншоты лендингов в `data/`;
   - прописать сайты конкурентов в `COMPETITOR_URLS`.

2. Запустить backend и протестировать:
   - `POST /analyzetext` — отправить тексты (описания товаров, услуг, УТП);
   - `POST /analyzeimage` — отправить скриншоты главных страниц/баннеров;
   - получить JSON‑аналитику по каждому конкуренту.

3. Настроить парсинг:
   - заполнить `COMPETITOR_URLS` в `.env`;
   - вызвать `POST /parsedemo` — получить анализ сайта + скриншот;
   - результат автоматически добавится в историю.

4. Посмотреть историю:
   - `GET /history` — увидеть краткие резюме по тексту и визуальной части.

5. (Опционально) Работать через desktop‑клиент:
   - собрать `competitionmonitor.exe` и запускать типичные сценарии прямо из GUI.

## Ограничения и известные проблемы

- **PyQt6 / Windows DISPLAY ошибка**  
  На моей машине при запуске `python desktop/main.py` или `dist/competitionmonitor.exe` возникает сообщение:

  ```text
  qt.qpa.screen: "Unable to open monitor interface to \\\\.\\DISPLAY1:" "Unknown error 0xe0000225."
  ```

  Это связано с особенностями окружения/драйверов дисплея, а не с логикой приложения.  
  Backend (FastAPI + Selenium + ProxyAPI) и все API‑эндпоинты (`/analyzetext`, `/analyzeimage`, `/parsedemo`, `/history`) работают корректно и протестированы через веб‑интерфейс.

- **Безопасность API‑ключей**  
  API‑ключи и другие чувствительные данные хранятся только в `.env`.  
  Файл `.env` добавлен в `.gitignore` и не входит в репозиторий.

- **In‑memory история**  
  История запросов хранится в памяти процесса (и/или в `history.json` как в заготовке), поэтому при перезапуске приложения история может обнуляться. Для production‑варианта можно подключить БД.
